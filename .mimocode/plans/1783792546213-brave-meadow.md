# Plan: WebSocket Listener & Online Status Emission

## Context

The backend already has a fully functional `NotificationsGateway` (socket.io v4) that:
- Authenticates WS connections via JWT in the `authorization` handshake header
- Tracks per-user socket count for online/offline detection
- Broadcasts `userStatusChanged` events (`USER_ONLINE` / `USER_OFFLINE`) to users who share conversations
- Emits real-time events: `message:new`, `message:edited`, `message:deleted`, `match:new`, `shoutout:received`, `shoutout:replied`, `notification:announcement`

The frontend has **zero** socket.io client code. This plan adds the client-side integration.

## Backend Summary (read-only reference)

- **Gateway**: `src/modules/notifications/notifications.gateway.ts` ‚Äî `@WebSocketGateway({ cors: { origin: '*' } })`
- **Auth guard**: `src/common/guards/ws-jwt-auth.guard.ts` ‚Äî reads `client.handshake.headers.authorization`
- **Events emitted to clients**:
  - `userStatusChanged` ‚Üí `{ type: 'USER_ONLINE' | 'USER_OFFLINE', userId }`
  - `message:new` ‚Üí `{ type: 'NEW_MESSAGE', conversationId, message }`
  - `message:edited` ‚Üí `{ type: 'MESSAGE_EDITED', conversationId, message }`
  - `message:deleted` ‚Üí `{ type: 'MESSAGE_DELETED', conversationId, messageId }`
  - `match:new` ‚Üí `{ type: 'NEW_MATCH', match }`
  - `shoutout:received` ‚Üí `{ type: 'SHOUTOUT_RECEIVED', senderId, shoutout }`
  - `shoutout:replied` ‚Üí `{ type: 'SHOUTOUT_REPLIED', conversationId, shoutout }`
  - `notification:announcement` ‚Üí `{ type: 'ANNOUNCEMENT', ... }`
- **WS messages the client can send**: `updateFcmToken`, `sendMessage`, `test`

## Frontend Changes

### 1. Install `socket.io-client`
```bash
pnpm add socket.io-client
```

### 2. Create `src/lib/socket.ts` ‚Äî singleton socket manager

A plain module (not a React component) that:
- Creates a single `io()` instance connected to the backend API URL
- Passes `authorization: Bearer <token>` in handshake headers
- Provides `getSocket()`, `connect(token)`, `disconnect()` functions
- Auto-connects when `connect(token)` is called
- Handles reconnection with exponential backoff (socket.io built-in)
- Exports the socket instance for direct `.on()` usage

Key details:
- URL: use `NEXT_PUBLIC_API_URL` (same as REST API base)
- Transport: `websocket` first, fallback to `polling`
- Reconnection: enabled by default, with `reconnectionAttempts: 10`, `reconnectionDelay: 1000`

### 3. Create `src/hooks/use-socket.ts` ‚Äî React hook for socket lifecycle

A hook that:
- Reads `accessToken` from `useAuthStore`
- On mount / token change: calls `connect(token)` if token exists
- On logout (token becomes null): calls `disconnect()`
- Returns `{ socket, isConnected }` for consuming components
- Cleans up on unmount

### 4. Integrate into `AuthProvider` (`src/components/auth-provider.tsx`)

Add the `useSocket` hook inside `AuthProvider` so the socket lifecycle is tied to auth state:
- Connect when user is authenticated (token exists)
- Disconnect when logged out
- Expose `socket` and `isConnected` via context

### 5. Create `src/hooks/use-user-status.ts` ‚Äî online status listener

A hook that:
- Subscribes to `userStatusChanged` events on the socket
- Maintains a local `Map<userId, { isOnline: boolean }>` state
- Provides `getOnlineStatus(userId)` function and `onlineUsers` set
- Cleans up listener on unmount
- Integrates with React Query cache to update `isOnline` on relevant query data (e.g., conversation list, visit profile)

### 6. Update `src/components/providers.tsx`

No changes needed ‚Äî the socket lives inside `AuthProvider` which is already nested in `Providers`.

### 7. Update conversation list to show online status

In `src/app/chats/_components/conversation.tsx`:
- Use `useUserStatus()` to show online dot indicator on conversation rows

### 8. Update profile sheet to show live online status

In `src/components/mobile/profile-sheet.tsx`:
- Use `useUserStatus()` to show real-time online badge on the visited profile
