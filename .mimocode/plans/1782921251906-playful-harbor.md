# Fix: Hide swipe action buttons in non-discovery ProfileSheet usage

## Problem
`ProfileSheet` (`src/components/mobile/profile-sheet.tsx:197-247`) always renders the bottom action bar (Pass / Shoutout / Like buttons). This bar is only meaningful in the **discover** context, but it also shows when viewing profiles from **chat/shoutout** screens where those actions are irrelevant (currently passed as no-ops `() => {}`).

## Usage sites
| Location | File | Props |
|----------|------|-------|
| Discover | `src/app/(web)/discover/_components/discover-client.tsx:334` | Real like/dislike/shoutout handlers |
| Chat list 1 | `src/app/(web)/chats/_components/chat-list.tsx:169` | No-op handlers + `onMessage` |
| Chat list 2 | `src/app/(web)/chats/_components/chat-list.tsx:354` | No-op handlers + `onMessage` |

## Plan

### 1. Add `showActions` prop to `ProfileSheet`
**File:** `src/components/mobile/profile-sheet.tsx`

- Add `showActions?: boolean` prop (defaults to `true` for backward compat)
- Wrap the bottom action bar (lines 197-247) with `{showActions && ...}`

### 2. Pass `showActions={false}` from chat contexts
**File:** `src/app/(web)/chats/_components/chat-list.tsx`

- At both `<ProfileSheet>` usages (lines 169, 354), add `showActions={false}`
- Clean up the now-unused no-op `onLike`, `onDislike`, `onShoutout` props (make them optional too)

### 3. Make like/dislike/shoutout props optional
**File:** `src/components/mobile/profile-sheet.tsx`

- Since chat contexts won't use them, make `onLike`, `onDislike`, `onShoutout` optional (`?:`) so callers don't need to pass no-ops

## Verification
- `npm run build` (or `next build`) should pass with no type errors
- In the app: discover profile sheet still shows action buttons; chat profile sheet does not
