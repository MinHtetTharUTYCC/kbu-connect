# Prefetch Profile on Dwell

## Goal
After a user views a discovery card for 3 seconds, prefetch the full profile via React Query so `ProfileSheet` opens instantly (no loading spinner).

## File to modify
`src/app/(web)/discover/_components/discover-client.tsx`

## Changes

1. **Add imports**: `useQueryClient` from `@tanstack/react-query`, `getUsersControllerGetUserProfileQueryOptions` from `@services/generated/users/users`

2. **Add a `useEffect`** after the existing prefetch effect (~line 50) that:
   - Runs when `profile?.id` changes
   - Sets a 3-second `setTimeout`
   - On timeout, calls `queryClient.prefetchQuery(getUsersControllerGetUserProfileQueryOptions(profile.id, { query: { staleTime: 1000 * 60 * 5 } }))`
   - Returns a cleanup function that clears the timeout
   - Guard: skip if `profile` is undefined

```typescript
const queryClient = useQueryClient();

useEffect(() => {
    if (!profile) return;
    const timer = window.setTimeout(() => {
        queryClient.prefetchQuery(
            getUsersControllerGetUserProfileQueryOptions(profile.id, {
                query: { staleTime: 1000 * 60 * 5 },
            }),
        );
    }, 3000);
    return () => clearTimeout(timer);
}, [queryClient, profile]);
```

## Why this works
- React Query caches the prefetched data for 5 minutes (staleTime)
- When `ProfileSheet` opens and calls `useVisitProfile(userId)`, it gets a cache hit → `isLoading` is false immediately
- If the user swipes before 3s, the cleanup clears the timer → no wasted request
- `queryClient.prefetchQuery` uses the same query key as `useVisitProfile`, so the cache is shared

## Verification
1. Open the discover page, stay on a card for 3+ seconds
2. Open React Query DevTools (if available) or check Network tab — a `GET /users/{id}` request should fire at ~3s
3. Tap the profile name → `ProfileSheet` should open instantly without a spinner
4. Swipe quickly (< 3s) to a new card → no network request fires for the previous profile
