# Plan: Add Real-Time Socket Event Listeners

## Overview
Add a single `useSocketEvents` hook that listens to all backend socket events and updates React Query caches. Add a `MatchCelebration` full-screen overlay (via `createPortal`) for new matches.

## Files to Create

### 1. `src/hooks/use-socket-events.ts`
Single hook that registers listeners for all 6 events and updates caches directly.

**Dependencies:** `useQueryClient()`, `useSocketContext()`, `useAuthContext()`, `usePathname()`, `useRouter()`

**Key pattern:** Use `useRef` for `activeConversationId` (parsed from pathname) and `myId` to avoid stale closures in event handlers.

**Event handlers:**

| Event | Active conversation? | Action |
|---|---|---|
| `message:new` | Yes | Replace temp message (dedup by senderId+content on `temp-*` IDs) or prepend to `pages[0].messages` |
| `message:new` | No | Move conversation to top of chats list, increment chat unread count, show toast |
| `message:edited` | Yes | Update message content by `id` across all pages |
| `message:deleted` | Yes | Filter out message by `messageId` from all pages |
| `match:new` | N/A | Prepend `MatchItemDto` to matches cache, dispatch `new-match` CustomEvent on `window` |
| `shoutout:received` | N/A | Parse JSON, prepend to received shoutouts cache, show toast |
| `shoutout:replied` | N/A | Parse JSON, remove shoutout by `id` from received list, prepend new conversation to chats list |

**Cache manipulation patterns** (reuse existing logic):
- Prepend to infinite page 0: follow `useSendMessage` pattern (`use-send-message.ts:26-39`)
- Replace temp message: follow `useSendMessage.replaceTempMessage` pattern (`use-send-message.ts:47-72`)
- Edit message in-place: follow `useEditMessage.onMutate` pattern (`use-edit-message.ts:16-25`)
- Delete message: filter by `messageId` from all pages
- Remove shoutout + insert conversation: follow `useReplyShoutout.onSuccess` pattern (`use-reply-shoutout.ts:23-51`)

**Query keys needed:**
- `getChatControllerGetConversationMessagesInfiniteQueryKey(conversationId)`
- `getChatControllerGetConversationsInfiniteQueryKey()`
- `getChatControllerGetShoutoutsInfiniteQueryKey()`
- `getChatControllerGetUnreadCountQueryKey()`
- `getMatchesControllerGetMatchesInfiniteQueryKey()`
- `getNotificationsControllerGetUnreadCountQueryKey()`

### 2. `src/components/match-celebration.tsx`

## Edge Cases
- **Duplicate messages**: Match `temp-*` IDs by `senderId` + `content`, replace instead of double-adding
- **Stale closures**: All event handlers read `activeConversationIdRef.current` and `myIdRef.current`
- **No user logged in**: Early return from hook if `user?.user?.id` is falsy
- **Cache not populated**: `setQueryData` updater handles `undefined` gracefully (early return)
- **Own messages in background**: Toast says "Message sent" instead of "New message from you"
- **shoutout:replied**: Backend creates the conversation — insert at top of chats list like `useReplyShoutout` does

## Verification
1. Open two browser tabs as different users
2. Send a message from tab A → tab B should see it appear instantly if viewing that conversation
3. Send a message to a conversation tab B is NOT viewing → tab B should see conversation move to top + toast
4. Edit/delete a message from tab A → tab B should see the change if viewing that conversation
5. Match two users → the newly matched user should see the full-screen celebration overlay
6. Send a shoutout → receiver should see it in list + toast
7. Reply to a shoutout → original sender should see shoutout removed + new conversation appear in chats
