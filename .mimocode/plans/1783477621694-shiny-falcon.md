# Add Click Handlers to Notification Rows

## Context
Notification rows currently have no click behavior. Each notification type should navigate to a different route or open a detail sheet on click.

---

## Files to Modify

| File | Change |
|---|---|
| `src/app/notifications/_components/notification-client.tsx` | Add `useRouter`, pass `onNavigate` to `NotificationRow`, handle click routing by type |
| `src/app/notifications/_components/notification-detail-sheet.tsx` | **New file** — Drawer-based bottom sheet for SYSTEM/ANNOUNCEMENT notifications showing title + body + close button |

---

## Implementation

### 1. Create `src/app/notifications/_components/notification-detail-sheet.tsx`

A simple bottom sheet for SYSTEM and ANNOUNCEMENT notifications:
- Uses `Drawer` + `DrawerContent` from `@/components/ui/drawer` (same pattern as ProfileSheet)
- Props: `notification: NotificationItemDto`, `onClose: () => void`
- Shows: notification icon, title, body, created date
- Close button in footer (or DrawerClose)
- `open` controlled by parent via `!!notification`

### 2. Update `NotificationClient`

- Add `useRouter()` 
- Add `selectedNotification` state for the detail sheet
- Pass an `onNavigate` callback to each `NotificationRow`:
  ```
  (notification) => {
    switch (notification.type) {
      case NEW_MESSAGE: router.push(`/chats/${notification.data?.conversationId}`); break;
      case NEW_MATCH: router.push('/matches'); break;
      case SHOUTOUT_RECEIVED: router.push('/chats?tab=shoutouts'); break;
      case SHOUTOUT_REPLIED: router.push(`/chats/${notification.data?.conversationId}`); break;
      case SYSTEM:
      case ANNOUNCEMENT: setSelectedNotification(notification); break;
    }
  }
  ```
- Render `<NotificationDetailSheet>` at the bottom when `selectedNotification` is set

### 3. Update `NotificationRow`

- Change outer `<div>` to `<button>` for accessibility
- Accept `onNavigate: (notification: NotificationItemDto) => void` prop
- Call `onNavigate(notification)` on click
- Add `active:bg-black/5` for press feedback

## Notification Type Routing

| Type | Action |
|---|---|
| `NEW_MESSAGE` | → `/chats/${data.conversationId}` |
| `NEW_MATCH` | → `/matches` |
| `SHOUTOUT_RECEIVED` | → `/chats?tab=shoutouts` |
| `SHOUTOUT_REPLIED` | → `/chats/${data.conversationId}` |
| `SYSTEM` | → Open detail sheet |
| `ANNOUNCEMENT` | → Open detail sheet |
